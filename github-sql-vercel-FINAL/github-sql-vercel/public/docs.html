Auth headers

Engine accepts native and Supabase-style headers:

x-api-key: Testplay
apikey: Testplay
Authorization: Bearer Testplay

Auth compatibility

Engine is API-key based today, but exposes a Supabase-style current user endpoint for client compatibility:

GET /api/auth/v1/user

REST table API

ActionEndpoint
GET select/api/rest/v1/users?select=id,name&email=eq.a@b.com&order=id.desc&limit=10
POST insert/api/rest/v1/users
PUT update/api/rest/v1/users?id=eq.1
DELETE delete/api/rest/v1/users?id=eq.1

The legacy /api/tables/:table endpoints remain available. The /api/rest/v1/:table path is the Supabase/PostgREST-style alias.

JavaScript client

<script type="module">
import { createClient } from '/engine-client.js';
const engine = createClient('https://your-project.vercel.app', 'Testplay');

await engine.sql('CREATE TABLE users (id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT, email TEXT)');
await engine.from('users').insert({ name: 'Asha', email: 'asha@example.com' });
const rows = await engine.from('users').select('id,name,email').eq('email', 'asha@example.com').get();
console.log(rows.data);
</script>

SQL admin endpoint

POST /api/query is admin/service-only. Use it for schema work and trusted maintenance, not public browser clients.

{ "sql": "CREATE TABLE users (id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT)" }

Buffered operations

REST writes update the in-memory SQLite buffer immediately and are flushed to GitHub after the configured delay. This keeps UX fast while reducing GitHub PAT/API write pressure.

GET /api/admin/operations
POST /api/admin/operations { "table": "users" }

OpenAPI

Fetch the machine-readable spec:

curl -H "x-api-key: Testplay" https://your-project.vercel.app/api/openapi

Windows CMD quickstart

curl -H "apikey: Testplay" https://your-project.vercel.app/api/init

curl -X POST https://your-project.vercel.app/api/query ^
  -H "Content-Type: application/json" ^
  -H "apikey: Testplay" ^
  -d "{\"sql\":\"CREATE TABLE users (id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT, email TEXT)\"}"

curl -X POST https://your-project.vercel.app/api/rest/v1/users ^
  -H "Content-Type: application/json" ^
  -H "apikey: Testplay" ^
  -d "{\"name\":\"Asha\",\"email\":\"asha@example.com\"}"

curl -H "apikey: Testplay" "https://your-project.vercel.app/api/rest/v1/users?select=id,name,email&order=id.desc&limit=10"
